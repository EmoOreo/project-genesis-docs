# UI_DIRECTION.md

**Last Updated:** May 28, 2026

## Core UI Identity

Project Genesis is evolving toward a scientific breeding terminal and sanctuary management interface.

The UI should feel like:
- scientific infrastructure
- biological specimen registry
- genetics analysis terminal
- sanctuary operations dashboard

Not:
- arcade HUD
- fantasy monster menu
- casual mobile UI

## Primary Inspirations

- RimWorld colonist panels
- Football Manager roster readability
- Planet Zoo genetics/specimen management
- Jurassic World Evolution facility management
- Ark-style breeding projection readability

## Current Direction

The Breeding Lab now supports predictive breeding readability through:
- projected offspring panel
- parent pairing context
- trait/stat projection
- mutation variance display
- lineage-aware breeding flow

## UI Priority

The next UI priority is no longer basic functionality.

The next priority is data readability:
- CreatureCard system
- registry scanability
- mutation highlighting
- color-coded traits
- stability/warning states
- visual hierarchy

## CreatureCard Philosophy

CreatureCard should become the reusable biological UI unit used across:
- Creature Roster
- Parent slots
- Offspring preview
- Habitat assignment
- Lineage viewer
- Observatory
- future expedition prep

Minimum CreatureCard information:
- icon/silhouette
- display name
- generation
- clade
- mutation count
- stability rating
- key traits
- compatibility/status indicator

## Visual Hierarchy Standards

Suggested color language:
- green = strong / positive
- yellow = average / caution
- red = weak / danger
- purple = mutation
- cyan = adaptive/scientific bonus

The UI should prioritize readable biological decision-making over decorative polish.
