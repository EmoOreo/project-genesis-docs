# Project Genesis Public Docs

Project Genesis is a single-player scientific creature genetics and sanctuary management simulation built in Godot 4.6.2 with C#.

The game centers on Sanctuary Site-04, a post-collapse research facility where the player breeds, studies, mutates, and preserves synthetic/hybrid organisms through a Dominant/Recessive/Mutation genetics system.

## Purpose of This Repository

This is the public documentation and AI-collaboration repository for Project Genesis.

It contains design, architecture, roadmap, lore, and planning documents only.

The actual Godot source code, scenes, scripts, and project files remain private.

## Current Project Identity

Project Genesis is evolving into a:

**scientific creature stewardship and genetics management simulation**

Core pillars:

- predictive breeding
- lineage and inheritance
- creature stewardship
- sanctuary operations
- biological experimentation
- scientific discovery
- long-term generational consequence

## Current Development Focus

The current priority is improving readability and management depth through:

- CreatureCard UI system
- registry UX improvements
- trait color coding
- mutation/stability indicators
- lineage viewer
- Habitat Dome systems
- save/load persistence

## AI Collaboration Instructions

If you are an AI model helping with Project Genesis:

1. Read `CURRENT_STATE.md`
2. Read `ARCHITECTURE.md`
3. Read `ACTIVE_TASKS.md`
4. Treat those as the authoritative project state.
5. Preserve SanctuarySite04 as the main game shell.
6. Preserve TestScene only as a debug/prototype scene.
7. Do not suggest full rewrites unless explicitly requested.
8. Prefer modular, Godot 4.6.2 C# compatible changes.
9. Recommend small implementation patches with clear file paths.
10. Keep the scientific breeding terminal identity intact.
